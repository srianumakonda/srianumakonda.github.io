Deep Learning has 3 main layers: an input layer, a hidden layer, and an output layer. The input layer is all about data. This is where you input your data into the neural network. From there, that data goes through different types of functions such as an activation function. Basically, let’s look at h[1][1] (the first neuron in the h1 layer in the image above ☝). h1 is the weighted sum of every single input in input layer i.

Think of it like this: let’s just say that i[1] (the first neuron in layer i) has a value of 1. The weight (which is the line that connects to the neurons in h1) has a value associated with it. Let’s say that the weight value for i[1] and h[1][1] is 2. Let’s say that we also have a bias of 1. So, we’d run this through our function Wx+b. We now just substitute the values ( 2*1 + 1 = 3) and now h[1][1] has a value of 3! We’d just continue doing this for every single neuron until we’re done.

Just to recap, the way we get x is from the input data. For example, each neuron in the input could be data in Fahrenheit and our goal is to teach the neural network how to convert that into Celsius. The w (weight) tells us how important that neuron is. The bigger the weight, the more prominent role it plays in the neural network. For example, if I wanted a computer to read images of handwritten digits, then we’d want the neurons where which store the handwritten digit to have a greater weight since that’s what we want to know.

After getting all the weighted sum for h[1][1], then we’d run that through an activation function. An activation function tells the neural network if a particular neuron should be fired or not. The output will always be between 0 and 1 (since the margin is small the computer will get higher accuracy). The output will vary based on the function you use. For example, the step function only outputs a 0 or 1 if the output is greater than a certain threshold. The rectified linear unit (ReLU) will return 0 if the output is 0 or less or else it’ll just return the output.

Getting into the Fun Stuff: Convolutional Neural Networks (CNNs)

ahhhhh, Convolutional Neural Networks. this is where the real fun beings. no, not fun as in watching your model training for hours, but fun as in spending hours and hours on stackoverflow over shape errors 😁. Anyway, you’re not here to hear me rant, so let’s dive right into it!

CNNs are a type of deep learning algorithm that specializes in image and object recognition. Most people will usually say, “Sri, what’s the difference between a CNN and a regular feedforward neural network?” Well, here’s the answer: while feedforward networks would work for basic images such as the MNIST dataset, they’ll be absolutely TERRIBLE at dealing with images that are so rich in data and pixels.

The overall goal of CNNs is to reduce the amount of “richness” the image has into a form that is really easy to process while maintaining the main features and details that are required to produce a really good prediction.

Chapter 1: Convolutions

Convolutional neural network — convolutions = neural network. Get it? Without convolutions, CNNs simply don’t exist. They “breathe” life into CNNs.

Convolutions work like this: we have something known as a kernel, a stride and padding. Let’s try breaking this down. You have a really complex math problem that your teacher told you to get done by tomorrow:

You’re not a madman to just substitute whatever value the teacher told you to and just solve it out. Instead, you use your 🧠 and realize that, “Hey! We can simply break this down until we get something really small that’s easier to deal with!”

That’s literally how convolutions work. The math problem is the image and the way you solve this is through your kernel, stride, and padding. Let’s talk a bit more about each:

The kernel (the green 3x3 grid above ☝) Is basically a filter that we pass through into our network. Using that, we’d do matrix multiplication over the whole image. The outputs we get is what the red grid on the right is. The amount of distance the kernel moves is known as the stride. In the example above, the stride is set to one which simply means that it moves over to the next matrix of pixels one at a time. If you had a stride of two, that’d mean you’d skip a row/column of pixels whenever doing this.

Once we do our matrix multiplication, we’d sum everything up and then output our single value (the values in red). The kernel values (which does the matrix multiplication with the pixels) are usually chosen randomly and then optimized using gradient descent. But sometimes, your image matrix’s length isn’t even. What if you have a 5x5 matrix instead of the 6 by 6 above? Wouldn’t the pixels overlap?

The answer to this is padding, specifically zero padding. The name gives it all; we add a border of zeros to the image.

Now, after doing this, our matrix will become even again!! Hopefully that was an intuitive example of what convolutions are. If you have any questions, dm me @srianumakonda.

Chapter 3: Dropout and Overfitting

Think of overfitting like this: let’s say that your teacher told you to learn your multiplication tables so that you can do EQAO. So, as a good student, you memorize everything. But then, when you write EQAO, they ask you to “explain your answer” and how 8 * 8 = 64. Well, you’re screwed. You just focused on memorizing and not understanding how 8 * 8 = 64.

That’s what overfitting is in neural networks. The neural network memorizes all the training data and when it gets given a testing image, it just breaks.

The way we solve this is through dropout. Dropout is a method that randomly drops “neurons”. This means that instead of having a fixed amount of neurons for each layer, we’re gonna have a layer with a random amount of nodes instead of following the x amount of nodes we defined.

Dropout sounded weird to me at first. What’s the point of turning off neurons? Wouldn’t that make the model worse since we’re gonna losing those details and features?

No. That’s not gonna happen. Why? Let’s say that we’re dealing with the MNIST dataset which is literally just a library of handwritten digits.

When we feed this into the neural network, what if the weights are greater on the black background than the white background? What if the model is paying more attention to the background and not the digit itself?

BOOM 💥. That’s where dropout comes in. By turning off those nodes, they’ll randomly reset and start focusing on the handwritten digit itself and not continue to play around with the background.

Chapter 4: Batch Normalization

Batch normalization is generally a topic most people don’t know much about. If nothing makes sense, this article and this article should help you a bit better.

Whenever we have image data, the first thing most people will do is normalize the data. What that basically means is that whenever we have an image (let’s take a grayscale image for this example), the pixel values are between 0 and 255 (0 being black 255 being white). This is a HUGE range for the computer since of course, the computer has to convert everything to binary and use more RAM. So, what we can do is change the range from 0 to 255 to 0–1. That'll make it so much easier on the computer and help yourself by not having to wait really long.

The same can be applied to neural networks. Even if we pass everything through our activation functions, at some point we’ll end up having a really large value. Normalizing that can make it a lot easier on the model, speed it up by allowing us to use a higher learning rate, and can reduce overfitting in some situations.

Chapter 5: Activation Functions

There are a BUNCH of activation functions. I already talked about this above but the TLDR of it is that activation functions tell the neural network whether certain nodes should be “turned on” or not.

The activation function that I used for my neural network is known as the Rectified Linear Unit (ReLU). ReLU is hands down the most common type of activation function used because a) it can be used almost anywhere in Deep Learning and b) it’s actually really simple.

The first thing to note is that ReLU is non-linear in nature so fitting non-linear data through the activation function shouldn’t be a problem. The thing with ReLU is that it can make the model a lot lighter. ReLU works like this: if the output of the neuron ≥ 0, then the neuron won’t be fired since it has a value of 0. Else, ReLU will simply return the output.

The benefit of this is that not many neurons will fire which is surprisingly really good. This is mainly because it’ll make the model a lot lighter which in turn, will be able to train a lot more faster and use less RAM. It’s also not complicated at all and can generally be used in any situation.

Chapter 6: Putting it all together

Now that we’ve talked about the main components in the neural network, let’s put it all together! You can see the overall model architecture below:

neural_network(
(layer1): Sequential(
(0): Conv2d(1, 64, kernel_size=(3, 3), stride=(1, 1))
(1): ReLU()
(2): BatchNorm2d(64, eps=1e-05, momentum=0.1, affine=True, track_running_stats=True)
(3): Dropout(p=0.2, inplace=False)
(4): MaxPool2d(kernel_size=(2, 2), stride=(2, 2), padding=0, dilation=1, ceil_mode=False)
)
(layer2): Sequential(
(0): Conv2d(64, 128, kernel_size=(3, 3), stride=(1, 1))
(1): ReLU()
(2): BatchNorm2d(128, eps=1e-05, momentum=0.1, affine=True, track_running_stats=True)
(3): MaxPool2d(kernel_size=(2, 2), stride=(2, 2), padding=0, dilation=1, ceil_mode=False)
(4): Dropout(p=0.25, inplace=False)
(5): ReLU()
)
(layer3): Sequential(
(0): Conv2d(128, 256, kernel_size=(3, 3), stride=(1, 1))
(1): ReLU()
(2): BatchNorm2d(256, eps=1e-05, momentum=0.1, affine=True, track_running_stats=True)
(3): Dropout(p=0.2, inplace=False)
(4): MaxPool2d(kernel_size=(2, 2), stride=(2, 2), padding=0, dilation=1, ceil_mode=False)
)
(layer4): Sequential(
(0): Conv2d(256, 512, kernel_size=(3, 3), stride=(1, 1))
(1): ReLU()
(2): BatchNorm2d(512, eps=1e-05, momentum=0.1, affine=True, track_running_stats=True)
(3): Dropout(p=0.2, inplace=False)
(4): MaxPool2d(kernel_size=(2, 2), stride=(2, 2), padding=0, dilation=1, ceil_mode=False)
)
(flatten): Flatten()
(dropout): Dropout(p=0.25, inplace=False)
(output): Linear(in_features=512, out_features=2, bias=True)
)

We have 4 layers with each layer having a convolution, ReLU, then a BatchNormalization, Dropout, and then Maxpooling. We repeat that 4 times and then flatten the output into a 1-D vector. From there, we put another Dropout layer and then put a dense layer as our output (from 512 nodes to 2).
